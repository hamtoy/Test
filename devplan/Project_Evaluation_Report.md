## 📋 프로젝트 개요

### 🎯 목표 및 비전

- **차세대 AI 기반 코딩 어시스턴트**: Google Gemini 모델을 활용하여 코드 품질 평가, 개선 제안, 그리고 자동 리팩토링을 수행하는 지능형 에이전트 시스템입니다.
- **완전한 풀스택 관측성**: 단순 CLI 도구를 넘어, **실시간 로그 스트리밍(`WebSockets`)**과 **시각적 설정 편집(`ConfigEditor`)**이 가능한 웹 대시보드를 제공하여 운영 편의성을 극대화합니다.
- **확장 가능한 아키텍처**: Pydantic v2 기반의 견고한 타입 시스템과 FastAPI/Vite로 구성된 모던 스택을 통해 엔터프라이즈 환경에도 유연하게 대응합니다.

### 👥 대상 사용자 및 주요 사용 시나리오

- **대상 사용자**: 시니어 개발자, DevOps 엔지니어, AI 파이프라인 관리자.
- **사용 시나리오**:
  - **코드 리뷰 자동화**: PR 생성 전 프로젝트 전반의 품질 및 테스트 커버리지 자동 분석.
  - **실시간 트러블슈팅**: 웹 대시보드의 **LogViewer**를 통해 실행 중인 에이전트의 내부 상태와 로그를 실시간 모니터링.
  - **직관적인 설정 관리**: 복잡한 YAML/JSON/Environment 설정 대신 **ConfigEditor UI**를 통해 직관적으로 파이프라인 조정.

### 🧭 에코시스템 내 포지션

- **Frontend-Backend Bridge**: Python 백엔드의 강력한 처리 능력과 React 프론트엔드의 상호작용성을 결합한 **하이브리드 AI 툴**의 표준을 제시합니다.
- **Operational Excellence**: `aiofiles`를 활용한 완전 비동기 I/O 처리와 `FileLock`을 통한 안전한 동시성 제어로 엔터프라이즈급 안정성을 확보했습니다.

## 🧩 주요 모듈별 상세 평가

### 1) 실행/CLI 레이어 (`src/main.py`, `src/cli.py`, `src/ui/`)

- **기능 완성도:** 메뉴 기반 실행 및 비대화형 자동화(`--non-interactive`) 모드가 완성도 있게 동작합니다. `setup_logging` 최적화로 초기화 속도가 개선되었습니다.
- **코드 품질:** `rich` 라이브러리를 활용한 감각적인 UI와 `Neo4j` 선택적 의존성 처리가 깔끔합니다.
- **에러 처리:** 초기화 실패 시 명확한 폴백 메시지를 제공합니다.
- **성능:** Lazy Import 적용으로 CLI 시작 시간이 매우 빠릅니다.
- **강점:** "최소 설치"와 "풀 옵션"의 경계가 명확하여 배포 유연성이 높습니다.
- **약점 / 리스크:** 레거시 스크립트(`scripts/`) 중 일부가 최신 `src` 구조를 반영하지 못할 가능성이 있어 점검이 필요합니다.

### 2) 에이전트/워크플로우 코어 (`src/agent/`, `src/workflow/`, `src/features/lats.py`)

- **기능 완성도:** LATS(자기교정) 모듈의 안정화로 복잡한 추론 작업의 성공률이 향상되었습니다.
- **코드 품질:** Pydantic v2 기반의 엄격한 타입 검증이 데이터 무결성을 보장합니다.
- **에러 처리:** 재시도(Retry) 및 토큰 예산 관리 로직이 정교하게 구현되어 있습니다.
- **성능:** 자식 노드 병렬 평가(`_evaluate_children_parallel`) 도입으로 탐색 속도가 획기적으로 개선되었습니다(OPT-2 완료).
- **강점:** 단위 테스트 커버리지가 높고, 워크플로우 상태 관리가 명확합니다.

### 3) LLM 통합 및 프롬프트 (`src/llm/`)

- **기능 완성도:** Gemini Flash 모델에 최적화된 프롬프트 엔지니어링이 적용되어 응답 속도와 정확도가 균형을 이룹니다.
- **코드 품질:** v3.0 API 로딩 방식이 표준화되어 유지보수가 용이합니다.
- **에러 처리:** API 키 누락이나 모델 할당량 초과 시 사용자에게 구체적인 해결책을 제시합니다.
- **강점:** 정적 분석과 커버리지 강제로 변경 안정성이 매우 높습니다.

### 4) 데이터·RAG/그래프 (`src/qa/`, `src/graph/`)

- **기능 완성도:** Neo4j 기반 RAG 파이프라인이 선택적 기능으로 완벽히 분리되었습니다.
- **성능:** 2-Tier 인덱싱과 비동기 드라이버 사용으로 검색 지연 시간을 최소화했습니다.
- **에러 처리:** DB 연결 실패 시 RAG 기능을 조용히 우회(Bypass)하여 전체 장애를 방지합니다.
- **강점:** 복잡한 그래프 로직이 일반 사용자에게는 숨겨져 있어 진입 장벽을 낮춥니다.

### 5) Web API 및 프론트엔드 (`src/web/`, `static/`)

- **기능 완성도:** `LogViewer`와 `ConfigEditor` 추가로 웹 대시보드의 효용성이 급격히 상승했습니다. 유닛 테스트 추가로 프론트엔드 안정성이 확보되었습니다.
- **코드 품질:** `APIRouter` 분리 및 `aiofiles` 도입으로 비동기 처리가 완벽해졌습니다.
- **성능:** 로그 스트리밍을 완전 비동기(`aiofiles`)로 전환하여 이벤트 루프 블로킹 문제를 해결했습니다(OPT-1 완료).
- **강점:** 프론트엔드(`Vite`+`React`)와 백엔드 간의 타입 정의 및 API 계약이 명확합니다.
- **약점 / 리스크:** `config_api`의 동시성 문제는 `FileLock` 도입으로 해결되었습니다(PROMPT-001 완료).

### 6) 운영 및 인프라 (`src/infra/`, `scripts/`)

- **기능 완성도:** `build_release.py`를 통해 프론트엔드 빌드 및 백엔드 통합 배포가 자동화되었습니다(PROMPT-003 완료).
- **코드 품질:** `telemetry` 모듈의 옵션 처리와 `file_lock` 유틸리티의 크로스 플랫폼 지원이 돋보입니다.
- **에러 처리:** 외부 의존성(Redis 등) 부재 시 로컬 인메모리 캐시로 자동 전환되는 로직이 견고합니다.
- **강점:** 상용 서비스 수준의 관측성(Observability) 기반을 갖추었습니다.
