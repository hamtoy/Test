## 1. 프로젝트 개요 및 비전

### 1.1 프로젝트 정의

`shining-quasar`는 지식 그래프(Knowledge Graph)와 대규모 언어 모델(LLM)을 결합하여 고도의 정확성과 맥락 이해력을 갖춘 **Graph RAG 기반 QA 시스템**입니다. 단순한 텍스트 검색을 넘어, 데이터 간의 관계를 활용한 심층적인 추론과 답변 생성을 목표로 합니다.

### 1.2 핵심 로드맵 및 목표

- **Graph RAG 고도화:** Neo4j를 활용한 엔티티/관계 기반 검색으로 환각(Hallucination) 최소화.
- **에이전트 기반 추론:** LATS(Language Agent Tree Search) 및 ReAct 패턴을 도입하여 복잡한 질의 해결 능력 강화.
- **운영 안정성 확보:** 철저한 타입 시스템 적용, 에러 핸들링, 모델 폴백(Fallback) 메커니즘 구축.
- **확장성 있는 아키텍처:** FastAPI 기반의 모듈식 백엔드와 유연한 파이프라인 설계.

## 3. 상세 기능별 평가

### 3.1 LLM & Graph RAG

- **기능 완성도:** Gemini 1.5 모델 연동 및 Rate Limit Fallback이 완벽하게 구현되어 안정성이 뛰어납니다. Neo4j 기반의 그래프 검색은 환각을 효과적으로 억제합니다.
- **코드 품질:** `src/llm` 및 `src/qa` 모듈이 잘 분리되어 있으며, 최근 타입 시스템 강화로 인해 견고해졌습니다.
- **강점:** 복잡한 질의에 대해서도 그래프 데이터와 벡터 검색을 혼합하여 높은 정확도를 제공합니다.
- **약점 / 리스크:** 대량의 그래프 데이터 처리 시 Neo4j 쿼리 최적화가 필요할 수 있습니다.

### 3.2 LATS 에이전트 (Language Agent Tree Search)

- **기능 완성도:** 트리 탐색 기반의 추론 및 자가 검증 로직이 구현되어, 단일 단계 추론보다 훨씬 정교한 답변을 생성합니다.
- **코드 품질:** `features/lats.py`의 구현은 상태 관리와 백트래킹 로직이 명확합니다.
- **에러 처리:** 단계별 실패 시 다른 경로를 탐색하거나 재시도하는 로직이 내장되어 있습니다.
- **강점:** "생각하는" 에이전트로서 복잡한 인과관계 추론에 강점을 보입니다.

### 3.3 백엔드 및 통합 (FastAPI)

- **기능 완성도:** REST API는 물론 SSE 스트리밍을 통해 실시간 답변 생성을 지원합니다. 라우터 구조가 도메인별로 잘 분리되어 있습니다.
- **성능:** 비동기(`async`/`await`) 처리가 전반적으로 적용되어 높은 동시성 처리가 가능합니다.
- **강점:** Pydantic 모델을 활용한 자동 검증 및 명세 생성(OpenAPI)이 우수합니다.

### 3.4 운영 및 안정성 (Ops & QA)

- **기능 완성도:** 최근 적용된 로그 로테이션(OPT-2)과 엄격한 타입 시스템(PROMPT-001/OPT-1)으로 운영 리스크가 크게 감소했습니다.
- **코드 품질:** `mypy` strict 모드 준수율이 매우 높으며, 유틸리티 함수들의 타입 안정성도 확보되었습니다.
- **약점 / 리스크:** 아직 텔레메트리(Telemetry) 대시보드가 통합되지 않아 실시간 모니터링 편의성은 개선 여지가 있습니다.
